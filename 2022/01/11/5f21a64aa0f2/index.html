面向对象之抽象——从具体到通用的思维跃迁

发布于 2022-01-11 分类：技术, 2022, Java基础 👁

前言

抽象是面向对象四大特性中最”玄”的一个。封装是藏细节，继承是复用代码，多态是灵活调用，那抽象是什么？抽象是从具体事物中提取共性，忽略非本质细节的能力。2022年我逐渐意识到，抽象能力直接决定了代码的可维护性和扩展性。

什么是抽象？

抽象有两种体现形式：

  1. 抽象类：用 abstract 修饰的类，可以包含抽象方法和具体方法
  2. 接口：用 interface 定义，JDK8之后可以有 default 方法和 static 方法

抽象类 vs 接口

这是面试高频题，也是实际开发中经常纠结的选择。

维度 抽象类 接口
关键字 abstract class interface
构造方法 可以有 不能有
成员变量 任意 只能是常量（public static final）
方法 抽象+具体 抽象+default+static
多继承 单继承 多实现
设计层面 is-a 关系，模板设计 can-do 关系，行为契约

选择原则

  • 有公共状态和模板逻辑 → 抽象类
  • 只定义行为契约，无状态 → 接口
  • 不确定？优先用接口，更灵活

总结

抽象的本质是提取共性、屏蔽差异。2022年我对抽象的理解经历了三个阶段：一开始觉得抽象没用（直接写具体类不好吗？），后来觉得到处都该抽象（什么都要搞个接口），最终找到了平衡——在需要扩展的地方抽象，在稳定的地方具体。判断是否需要抽象的简单标准：当你写第二遍类似代码时，就该考虑抽象了。
