新手阶段写代码，字符串拼接我只会用 +，循环里也是一顿 + 操作，直到有一次代码审查被指出性能问题，才知道字符串拼接里面大有学问。String、StringBuilder、StringBuffer 三者的区别和适用场景，是Java开发者必须掌握的基础知识。

String：不可变的真相

String 对象一旦创建，内容不可修改。每次”修改”实际上是创建了一个新对象。

为什么String要设计成不可变？

  1. 字符串常量池：相同内容共享同一对象，节省内存
  2. 线程安全：不可变对象天生线程安全
  3. 安全性：String常作为参数（类名、文件路径、网络连接），不可变保证不被篡改
  4. hashCode缓存：不可变所以hash值可以缓存，提升HashMap等性能

扩容机制

StringBuilder 底层是 char[]（JDK9后是 byte[]），初始容量默认16：

扩容规则：当容量不足时，新容量 = (旧容量 + 1) * 2，如果还不够，直接用需要的容量。

最佳实践：如果能预估最终长度，初始化时指定容量，避免多次扩容。

什么时候用StringBuffer？

几乎不用。因为：

  1. 单线程场景：StringBuilder更快，没必要加锁
  2. 多线程场景：通常不会在多线程中共享同一个StringBuilder/StringBuffer，而是用ThreadLocal或局部变量
  3. 真正的线程安全字符串拼接：用 StringJoiner、Collectors.joining() 等更优雅

总结

场景 选择
少量拼接（编译期可确定） String +
单线程大量拼接 StringBuilder
多线程大量拼接（极少） StringBuffer
