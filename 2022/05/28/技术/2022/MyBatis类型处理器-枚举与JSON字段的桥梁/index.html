MyBatis类型处理器——枚举与JSON字段的桥梁

发布于 2022-05-28 分类：技术, 2022, MyBatis 👁

前言

数据库里存的是int和varchar，Java里用的是枚举和JSON对象，两者之间怎么映射？最初我在代码里手动转换，到处都是switch-case和JSON.parse，又丑又容易出错。直到学会TypeHandler，才知道MyBatis早就提供了优雅的解决方案。

TypeHandler是什么？

TypeHandler是MyBatis中Java类型与JDBC类型之间的转换器：

每次MyBatis执行SQL时，参数的设置和结果的读取都要经过TypeHandler。

内置TypeHandler

MyBatis内置了大量TypeHandler，覆盖了常见类型：

Java类型 JDBC类型 TypeHandler
String VARCHAR StringTypeHandler
Integer INTEGER IntegerTypeHandler
Long BIGINT LongTypeHandler
LocalDateTime TIMESTAMP LocalDateTimeTypeHandler
BigDecimal DECIMAL BigDecimalTypeHandler
byte[] BLOB ByteArrayTypeHandler

大部分情况下不需要关心TypeHandler，MyBatis会自动选择。

最佳实践

  1. 枚举优先用@EnumValue：MyBatis-Plus原生支持，最简单
  2. JSON字段用自定义TypeHandler：数据库存JSON，Java用对象
  3. autoResultMap = true：使用TypeHandler时必须开启
  4. 枚举统一规范：所有枚举都有getCode方法，方便通用处理
  5. TypeHandler注册到全局：避免每个字段都指定

总结

TypeHandler是MyBatis中容易被忽略但非常实用的功能。2022年我最大的收获是：不要在业务代码中手动转换枚举和JSON，交给TypeHandler统一处理。枚举用@EnumValue，JSON用自定义TypeHandler，代码更简洁、更安全。
