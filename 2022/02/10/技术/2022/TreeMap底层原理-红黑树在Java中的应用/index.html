TreeMap底层原理——红黑树在Java中的应用

发布于 2022-02-10 分类：技术, 2022, Java基础 👁

前言

第一次用TreeMap是在做一个排行榜功能，需要按分数排序展示用户排名。当时只知道TreeMap能自动排序，至于为什么能排序、底层怎么实现的，完全不清楚。后来面试被问到”TreeMap底层是什么”，我才开始认真研究红黑树。

TreeMap是什么？

TreeMap是基于红黑树（Red-Black Tree）实现的有序Map，键值对按照key的自然顺序或自定义顺序排列。

核心特点：

  • 有序：按key排序（自然顺序或Comparator指定顺序）
  • 不允许null key（需要比较，null无法比较）
  • 允许null value
  • 线程不安全
  • 增删查时间复杂度O(log n)

红黑树基础

红黑树是一种自平衡二叉搜索树，通过着色规则保证树的平衡：

五大规则：

  1. 每个节点非红即黑
  2. 根节点是黑色
  3. 叶子节点（NIL）是黑色
  4. 红色节点的子节点必须是黑色（不能有连续红节点）
  5. 从任一节点到其所有叶子节点的路径，黑色节点数量相同

TreeMap vs HashMap

特性 TreeMap HashMap
有序性 有序（key排序） 无序
null key 不允许 允许一个
时间复杂度 O(log n) O(1)
底层结构 红黑树 数组+链表+红黑树
内存占用 较少（无数组） 较多（数组+链表）
适用场景 需要排序/范围查询 通用键值存储

最佳实践

  1. 需要排序或范围查询时才用TreeMap，否则用HashMap
  2. 自定义key必须实现Comparable或提供Comparator
  3. 使用不可变对象作为key，避免排序错乱
  4. 善用NavigableMap的导航方法：firstEntry、lastEntry、higherEntry、lowerEntry等

总结

TreeMap的核心是红黑树，保证了有序性和O(log n)的操作效率。2022年我学到的最重要的一点是：TreeMap不是用来替代HashMap的，而是在需要排序和范围查询时的专用工具。日常开发中HashMap用得更多，但排行榜、区间查询、定时调度这些场景，TreeMap是最佳选择。
