面试高频问题

1. 为什么MySQL用B+树不用B树？

B+树非叶子节点不存数据，能存更多索引，树更矮，IO更少；叶子节点链表连接，范围查询高效；查询性能稳定。

2. 什么是回表？如何避免？

二级索引查到主键后再查聚簇索引获取完整数据。避免方式：覆盖索引（查询列都在索引中）。

3. 最左前缀原则是什么？

联合索引按定义顺序从左到右匹配，遇到范围查询(>、<、LIKE)后停止匹配。索引列顺序影响查询优化。

最佳实践

  1. EXPLAIN每一条慢SQL：确认索引是否生效
  2. 覆盖索引减少回表：查询列尽量在索引中
  3. 联合索引注意顺序：高区分度在前，等值查询在前
  4. 避免索引失效场景：函数、隐式转换、前缀LIKE
  5. 深分页用延迟关联：避免扫描大量数据

总结

索引是MySQL优化的核心。2023年深入B+树底层后，核心认知是：索引的本质是一种有序数据结构，理解了B+树的排序规则，就能推导出最左前缀原则、索引失效场景、覆盖索引原理。不再死记硬背索引规则，而是从底层原理出发分析问题。
