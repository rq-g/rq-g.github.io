面试高频问题

1. 如何优化慢SQL？

定位慢SQL → EXPLAIN分析 → 找出问题（全表扫描/索引失效/filesort）→ 制定方案（加索引/改写SQL/预计算）→ 验证效果。

2. 索引失效的场景有哪些？

函数操作、隐式类型转换、LIKE前缀通配符、OR连接非索引列、违反最左前缀、范围查询后的列。

3. 深分页怎么优化？

延迟关联（子查询走覆盖索引再回表）或游标分页（WHERE id > lastId）。

最佳实践

  1. 开启慢查询日志：线上必须开启
  2. EXPLAIN每一条慢SQL：确认索引是否生效
  3. 覆盖索引减少回表：查询列尽量在索引中
  4. 深分页用延迟关联：避免扫描大量数据
  5. 报表用预计算：统计表+定时任务

总结

慢查询优化是后端开发的核心能力。2023年形成方法论后，核心认知是：发现→分析→优化→验证，四步闭环。慢查询不是加索引就完事，而是要理解执行计划、分析根因、选择最优方案。
