2023年度总结——脱离CRUD内卷，进阶启航

发布于 2023-12-31 分类：技术, 2023, 年度总结 👁

前言

2023年是我技术进阶的关键一年。这一年我不仅深入学习了源码和中间件，更重要的是：参与了智慧矿山项目的技术架构设计，从系统设计到技术选型，从性能优化到问题排查，积累了宝贵的工业互联网项目实战经验。

年度项目经历

智慧矿山项目

项目背景：

  • 项目名称：某智慧矿山综合管控平台
  • 项目规模：覆盖10个矿区，10000+设备，5000用户
  • 我的职责：负责数据采集模块和监控告警模块的技术架构
  • 技术栈：SpringBoot + Redis + RocketMQ + Nacos + TDengine

核心成果：

  1. 数据采集模块

    • 实现矿区设备数据实时采集
    • 日均处理数据量：100万条
    • 数据采集延迟：<100ms
    • 技术亮点：
      • 使用TDengine存储时序数据，查询性能提升50倍
      • 使用RocketMQ异步处理，吞吐量提升10倍
      • 使用Nacos配置中心，实现动态配置管理

  2. 监控告警模块

    • 实现设备状态实时监控
    • 支持多级告警规则配置
    • 告警响应时间：<1秒
    • 技术亮点：
      • 使用Redis缓存告警规则，查询性能提升20倍
      • 使用Sentinel限流熔断，保护系统稳定性

年度踩坑与成长

1. 事务失效导致数据不一致

在项目中，转账接口部分数据未回滚。排查后发现是同类方法调用导致@Transactional失效。这次事故让我深入Spring事务源码，彻底搞懂了事务失效的所有场景。

2. Redis缓存雪崩

大促期间大量热点key同时过期，请求全部打到数据库。紧急加了随机过期时间和互斥锁方案。

3. 数据采集性能瓶颈

高峰期数据采集延迟超过5秒。通过引入TDengine和RocketMQ，采集延迟降低到100ms。

4. 系统高可用问题

单节点部署无法保证高可用。通过引入Nacos和Sentinel，系统可用性提升到99.9%。

年度关键认知

从”黑盒使用”到”源码理解”

2023年最大的转变不是记住了多少源码细节，而是遇到问题能从源码层面分析根因。事务失效看AOP代理、循环依赖看三级缓存、自动配置看SPI+条件注解——每个问题都有源码级别的解释。

从”被动排查”到”主动优化”

遇到问题只会百度，2023年能独立排查线上问题。接口超时看慢查询+线程堆栈，缓存异常看三大问题+持久化，消息丢失看Confirm+ACK，CPU飙高看GC日志+线程分析。

从”CRUD搬砖”到”方案设计”

能根据原型写代码，2023年可以独立负责业务模块的技术方案设计。缓存方案选型、消息队列落地、并发控制策略——不再是被动接收需求，而是主动思考技术方案。

2024年展望

2023年完成了进阶，2024年要走向高级：

技术深耕方向

  1. 微服务架构：Spring Cloud全链路、服务网格、分布式事务
  2. 高并发设计：限流熔断、缓存架构、读写分离、分库分表
  3. 性能调优：全链路压测、JVM调优、MySQL调优、系统调优
  4. 架构设计：DDD领域驱动、CQRS、事件溯源
  5. 云原生：Kubernetes、Service Mesh、Serverless

项目展望

2024年将参与核心项目：

  • 智慧矿山无人驾驶系统：50台无人驾驶矿卡
  • 技术挑战：高精度定位、实时控制、海量数据处理

给中级开发者的建议

  1. 源码比API重要：理解底层才能举一反三
  2. 原理比方案重要：理解原理才能选对方案
  3. 排查比解决重要：会排查才能独立解决
  4. 设计比编码重要：会设计才能脱离CRUD
  5. 思考比执行重要：会思考才能主动优化

结语
