Spring Cloud Gateway网关——微服务的统一入口

发布于 2023-04-30 分类：技术, 2023, 微服务 👁

前言

微服务拆分后，前端要调用5个不同服务的接口，每个服务都有自己的地址和认证方式。更麻烦的是，新增一个服务就要前端改一次配置。引入Spring Cloud Gateway作为统一入口后，前端只需要对接网关，所有请求统一路由、认证、限流。

生产环境注意事项

注意项 说明
不要引入Web依赖 Gateway基于WebFlux，与WebMVC冲突
路由配置持久化 推荐存储在Nacos配置中心
限流Key选择 根据业务选择IP/用户/接口
超时配置 合理设置connect-timeout和response-timeout
灰度发布 利用Weight断言实现流量分配
监控告警 接入Sentinel监控网关流量

总结

Spring Cloud Gateway是微服务的统一入口：路由转发让前端只需对接一个地址，过滤器链实现认证、限流、日志等横切关注点，负载均衡让服务实例对前端透明。网关不是可有可无的组件，而是微服务架构的标配——没有网关的微服务，就像没有大门的小区，谁都能进来，出了问题谁都不知道。
