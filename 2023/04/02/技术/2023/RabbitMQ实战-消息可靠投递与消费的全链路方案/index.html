面试高频问题

1. 如何保证消息不丢失？

生产者Confirm + 消息持久化 + 手动ACK，三个环节都保证。

2. 如何保证消息不重复消费？

唯一ID + 去重表 / 业务状态机 / Redis SETNX。

3. RabbitMQ如何实现延迟队列？

TTL + 死信队列，或使用rabbitmq_delayed_message_exchange插件。

最佳实践

  1. 生产环境必须手动ACK：自动ACK消息可能丢失
  2. 消息必须持久化：durable队列 + PERSISTENT消息
  3. 必须处理死信：消费失败的消息不能直接丢弃
  4. 幂等是必须的：MQ的语义是至少一次投递
  5. 监控消息堆积：设置告警阈值

总结

RabbitMQ的可靠消息投递是一个全链路问题。2023年我建立了完整的认知：生产者Confirm保证消息到达Broker，持久化保证Broker不丢消息，手动ACK保证消费不丢失，死信队列兜底异常消息，幂等机制防止重复消费。每一环都不能少。
