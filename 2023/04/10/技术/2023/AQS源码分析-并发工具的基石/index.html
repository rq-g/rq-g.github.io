面试高频问题

1. AQS的原理？

AQS通过volatile int state + CLH双向队列实现同步。state表示同步状态，队列存放等待线程。子类实现tryAcquire/tryRelease定义获取释放逻辑。

2. AQS的公平和非公平怎么实现？

公平锁在tryAcquire中先检查hasQueuedPredecessors()，队列有等待线程就不插队。非公平锁直接CAS竞争state。

3. 独占模式和共享模式的区别？

独占模式同一时刻只有一个线程获取锁（ReentrantLock），共享模式允许多个线程同时获取（Semaphore、CountDownLatch）。共享模式获取成功后会传播唤醒后续共享节点。

最佳实践

  1. 理解AQS才能理解JUC：它是ReentrantLock、Semaphore等的底层
  2. state的含义因工具而异：锁计数、许可数、倒计数
  3. CLH队列保证公平性：先到先得
  4. CAS+park/unpark：高效的自旋+阻塞机制
  5. 模板方法模式：AQS定义框架，子类实现细节

总结

AQS是JUC并发包的基石。2023年深入源码后，核心认知是：AQS = volatile state + CLH队列 + CAS + park/unpark。理解了AQS，ReentrantLock、Semaphore、CountDownLatch的原理就都通了——它们只是AQS的不同使用方式。
