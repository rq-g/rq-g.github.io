==与equals面试题——HashMap为什么必须重写hashCode

发布于 2026-01-01 分类：技术, 2026, Java基础, 面试题 👁

前言

2022年我写了一段代码，把自定义对象作为HashMap的key，结果get()死活拿不到值，debug了半天才发现忘记重写equals和hashCode。这个坑踩过一次就记住了——HashMap的key必须同时重写这两个方法。

1. == 运算符

四、面试高频问题

1. 为什么要有hashCode

提高查找效率。先用hashCode定位桶（O(1)），再在桶内用equals比较（O(n)或O(logn)）。如果没有hashCode，每次都要遍历所有元素比较equals，效率太低。

2. 两个对象hashCode相同，equals一定相同吗。

*不一、。hash冲突是正常的，HashMap用链表或红黑树处理冲突。

3. equals和hashCode可以用不同的字段吗？

*最好不。。如果equals用id比较，hashCode用name，可能出现equals相等但hashCode不等的情况，违反黄金法则。

4. 可变对象作为HashMap的key要注意什么？

最好不要用可变对象做key。如果key的属性变了，hashCode也会变，get()就找不到了。

// →可变对象做key
List<String> key = new ArrayList<>();
key.add("a");
Map<List<String>, String> map = new HashMap<>();
map.put(key, "value");

key.add("b");  // hashCode变了。map.get(key);  // 可能找不。```
