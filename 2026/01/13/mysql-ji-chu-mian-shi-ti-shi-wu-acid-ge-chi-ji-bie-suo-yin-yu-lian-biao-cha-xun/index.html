MySQL是Java后端开发必备的数据库，面试时关于MySQL的问题几乎必问题022年我因为不了解事务隔离级别，在一次线上问题排查中走了很多弯路。后来专门系统学习了MySQL的事务和索引知识，这篇文章整理了最核心的MySQL面试题。

一、事务四大特。ACID

1. ACID 详解

特。 全称 说明 示例
Atomicity 原子。 事务是一个不可分割的工作单元，要么全部执行，要么全部不执行 转账：要么扣款和收款都成功，要么都失。
Consistency 一致。 事务执行前后，数据库的完整性约束没有被破坏 转账前后总金额不。
Isolation 隔离。 多个事务并发执行时，彼此之间不能互相干扰 事务A看不到事务B未提交的数据
Durability 持久。 事务一旦提交，对数据库的修改就是永久性的 断电后数据仍然存。

2. 各隔离级别详情

READ UNCOMMITTED（读未提交）

SERIALIZABLE（串行化工

特点：最高隔离级别，完全串行执行，性能最差。

3. MySQL 默认隔离级别

MySQL默认是REPEATABLE READ（可重复读），通过MVCC（多版本并发控制）解决了不可重复读问题，但仍可能发生幻读。

六、面试高频问题

1. 什么是幻读？MySQL如何解决幻读。

答案：幻读是指同一事务内两次查询，第二次查询出现了第一次查询没有的行。MySQL在可重复读级别下通过MVCC和间隙锁解决幻读问题。

2. 索引为什么使用B+树？

答案 - B+树是平衡树，查询效率稳定O(logn)

  • 叶子节点存储数据（聚簇索引）或指向数据的指针
  • 叶子节点形成链表，便于范围查询- 相比B树，B+树减少了磁盘IO次数

3. 什么是覆盖索引。

答案：覆盖索引是指索引包含了查询所需的所有字段，不需要回表查询。例如SELECT id, name FROM users WHERE name = '张三'，如果name是索引，就不需要回表。

4. 如何优化慢SQL?

答案 - 添加合适的索引

  • 避免SELECT *
  • 避免在索引列上做计算
  • 优化联表查询
  • 使用EXPLAIN分析执行计划

总结

MySQL的核心知识点包括。1. 事务ACID：原子性、一致性、隔离性、持久。2. 隔离级别：四个级别，MySQL默认可重复读
3. 索引：主键索引（聚簇）vs 普通索引（非聚簇）
4. 联表查询：INNER JOIN、LEFT JOIN、RIGHT JOIN的区别
掌握这些知识，面试时就能应对大部分MySQL问题了。
