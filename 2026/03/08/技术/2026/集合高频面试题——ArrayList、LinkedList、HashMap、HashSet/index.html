集合高频面试题——ArrayList、LinkedList、HashMap、HashSet

发布于 2026-03-08 分类：技术, 2026, Java基础, 面试题 👁

前言

Java集合是面试的重中之重，ArrayList、LinkedList、HashMap、HashSet这四个类，每次面试必问题022年我把HashMap源码啃了三遍，从”只会。→能讲清楚底层原理”，这个过程让我对数据结构有了深刻理解决

一、ArrayList 和 LinkedList 的区别

1. 底层数据结构

ArrayList：动态数据

LinkedList：双向链路

2. 核心区别对比

维度 ArrayList LinkedList
底层结构 动态数据 双向链表
随机访问 O(1) O(n)
头部插入 O(n) O(1)
中间插入 O(n) O(n)
尾部插入 O(1) 快（不扩容） O(1)
删除头部 O(n) O(1)
内存占用 连续内存（有预留空间。 分散内存（每个节点多两个指针。

3. 效率对比

  1. 计算hash：hash = (h = key.hashCode()) ^ (h >>> 16)
  2. 定位桶：index = (n - 1) & hash

*为什么容量必须是2的幂等

  • (n - 1) & hash 等价。hash % n，但位运算更。- 扩容时元素迁移更高效

扩容后可能变成：A 和 B A（成环）
get()时遍历链路死循。```

JDK1.8改进：改用尾插法，避免了死循环，但仍然不是线程安全的。

问题3：size 不准。

集合 适用场景 注意事项
ArrayList 随机查询、尾部插。 指定初始容量避免扩容
LinkedList 队列/栈、头部插。 尽量少用随机访问
HashMap 键值对存储 重写key的equals和hashCode
ConcurrentHashMap 并发场景 不能用null key
HashSet 去重 元素要重写equals和hashCode

总结

集合是Java开发的基本功，面试时要做到。1. ArrayList/LinkedList：能说出底层结构和效率对账2. HashMap：能讲清楚JDK1.7..8的区别、put流程、扩容机制3. HashSet：知道底层是HashMap，以及为什么要重写equals和hashCode
4. 线程安全：知道HashMap为什么不安全，以及用什么替。
这些知识不仅面试有用，日常开发中也能帮你写出更高效的代码。
