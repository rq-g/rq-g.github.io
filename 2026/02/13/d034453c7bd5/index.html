五、Spring 事务传播机制

1. 七种传播机制 | 传播属。| 说明 | |———-|——| | REQUIRED | 默认，当前有事务就加入，没有就新。| | REQUIRES_NEW | 新建事务，挂起当前事务| | SUPPORTS | 当前有事务就加入，没有就以非事务方式执行 | | NOT_SUPPORTED | 以非事务方式执行，挂起当前事务| | MANDATORY | 当前必须有事务，否则抛出异常 | | NEVER | 当前不能有事务，否则抛出异常 | | NESTED | 如果当前有事务，嵌套事务执行 | ### 2. REQUIRED vs REQUIRES_NEW

3. 事务失效场景 | 场景 | 说明 | 示例 | |——|——|——| | 方法不是public | @Transactional只对public方法生效 | private void save() | | 内部方法调用 | 绕过了代理对账| this.save() | | 异常被捕。| 没有抛出异常 | try { ... } catch (Exception e) {} | | 抛出非检查异。| 默认只回滚RuntimeException | 抛出Exception | | 配置错误 | propagation配置错误 | SUPPORTS在无事务环境 | | 数据库不支持 | 如MyISAM不支持事务| 存储引擎错误 | —

总结

Spring源码是中级面试的重点，掌握这些知识不仅能应对面试，还能更好地理解Spring框架的设计思想。核心要点：

  1. IOC流程：refresh()方法是核心，包含BeanDefinition注册和Bean生命周期管理
  2. AOP原理：基于动态代理，JDK代理接口，CGLIB代理。3. 事务传播：七种传播机制，重点掌握REQUIRED和REQUIRES_NEW
  3. 常见坑点：内部调用绕过代理、异常被捕获导致事务不回滚、单例Bean线程安全

这些都是面试中高频问到的问题，一定要深入理解决
