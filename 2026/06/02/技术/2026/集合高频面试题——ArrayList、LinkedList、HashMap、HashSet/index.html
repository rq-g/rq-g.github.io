集合高频面试题——ArrayList、LinkedList、HashMap、HashSet

发布于 2026-06-02 分类：技术, 2026, Java基础, 面试题 👁

前言

Java集合是面试的重中之重，ArrayList、LinkedList、HashMap、HashSet这四个类，每次面试必问。2022年我把HashMap源码啃了三遍，从”只会用”到”能讲清楚底层原理”，这个过程让我对数据结构有了深刻理解。

一、ArrayList 和 LinkedList 的区别

1. 底层数据结构

ArrayList：动态数组

LinkedList：双向链表

2. 核心区别对比

维度 ArrayList LinkedList
底层结构 动态数组 双向链表
随机访问 O(1) 快 O(n) 慢
头部插入 O(n) 慢 O(1) 快
中间插入 O(n) 慢 O(n) 慢
尾部插入 O(1) 快（不扩容） O(1) 快
删除头部 O(n) 慢 O(1) 快
内存占用 连续内存（有预留空间） 分散内存（每个节点多两个指针）

3. 效率对比

4. 业务场景选择

场景 选择 理由
主要是查询 ArrayList 随机访问O(1)
主要是头部插入/删除 LinkedList 链表操作O(1)
主要是尾部插入 ArrayList 效率高，内存连续
需要作为队列/栈 LinkedList 实现了Deque接口

注意：实际开发中90%以上的场景用ArrayList就够了，LinkedList的使用场景很少。

二、HashMap 底层结构

1. JDK1.7 vs JDK1.8

维度 JDK1.7 JDK1.8
数据结构 数组 + 链表 数组 + 链表 + 红黑树
插入方式 头插法 尾插法
扩容迁移 重新计算hash 原位置 或 原位置+旧容量
hash计算 9次扰动 2次扰动
死循环问题 有（并发扩容） 无

三、HashMap 为什么线程不安全

1. 并发场景下的问题

问题1：数据丢失

原因：size++不是原子操作，多个线程同时put可能覆盖。

问题2：JDK1.7 死循环

JDK1.8改进：改用尾插法，避免了死循环，但仍然不是线程安全的。

问题3：size 不准确

3. ConcurrentHashMap 原理（JDK1.8）

  • 数组 + 链表 + 红黑树
  • CAS + synchronized 保证线程安全

五、面试高频问题

1. ArrayList 怎么扩容？

2. HashMap 可以用 null 作为 key 吗？

可以。HashMap允许一个null key，放在index=0的位置。ConcurrentHashMap不允许null key。

3. HashSet 怎么保证有序？

HashSet无序。如果需要有序，用：

  • LinkedHashSet：保证插入顺序
  • TreeSet：保证自然顺序或自定义排序

六、最佳实践

集合 适用场景 注意事项
ArrayList 随机查询、尾部插入 指定初始容量避免扩容
LinkedList 队列/栈、头部插入 尽量少用随机访问
HashMap 键值对存储 重写key的equals和hashCode
ConcurrentHashMap 并发场景 不能用null key
HashSet 去重 元素要重写equals和hashCode

总结

集合是Java开发的基本功，面试时要做到：

  1. ArrayList/LinkedList：能说出底层结构和效率对比
  2. HashMap：能讲清楚JDK1.7和1.8的区别、put流程、扩容机制
  3. HashSet：知道底层是HashMap，以及为什么要重写equals和hashCode
  4. 线程安全：知道HashMap为什么不安全，以及用什么替代

这些知识不仅面试有用，日常开发中也能帮你写出更高效的代码。
