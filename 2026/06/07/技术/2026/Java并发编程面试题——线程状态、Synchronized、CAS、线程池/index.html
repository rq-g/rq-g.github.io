前言

并发编程是中级Java开发的分水岭，能熟练运用并发工具、理解底层原理的开发者，才能应对高并发场景。2023年我在项目中遇到过线程池配置不当导致的生产事故，后来专门系统学习了并发编程知识，这篇文章整理了最核心的并发面试题。

状态 说明 进入方式 退出方式
NEW 线程已创建但未启动 new Thread() start()
RUNNABLE 可运行状态 start() 获取锁失败→BLOCKED
BLOCKED 阻塞等待锁 等待synchronized锁 获取到锁
WAITING 无限期等待 wait()/join()/LockSupport.park() notify()/notifyAll()
TIMED_WAITING 限时等待 sleep(time)/wait(time)/join(time) 超时或被唤醒
TERMINATED 线程终止 run()执行完毕 无

二、Synchronized 和 Lock 的区别

1. 核心区别

维度 Synchronized Lock
性质 关键字（JVM层面） 接口（API层面）
锁获取 自动获取 手动获取（lock()）
锁释放 自动释放（异常也释放） 手动释放（unlock()）
等待中断 不可中断 可中断（lockInterruptibly()）
超时获取 不支持 支持（tryLock(time)）
公平锁 非公平 可配置公平/非公平
条件变量 不支持 支持（Condition）
性能 低（重量级锁） 高（CAS+自旋）

总结

并发编程是中级面试的重点，核心要点：

  1. 线程状态：六种状态及其转换
  2. 同步机制：Synchronized（JVM层面）vs Lock（API层面）
  3. volatile：保证可见性和禁止重排序，不保证原子性
  4. CAS：乐观锁，无锁并发的基础
  5. 线程池：七大参数、四种拒绝策略、工作流程
  6. 实战经验：禁止手动new Thread、线程池配置、死锁排查

掌握这些知识，才能在高并发场景下写出安全高效的代码。
