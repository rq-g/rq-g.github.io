一、Maven 依赖管理

1. 依赖传递

原理

Maven依赖具有传递性，当你引入一个依赖时，它的依赖也会被自动引入。

依赖传递规则

规则 说明
最短路径优先 路径短的依赖优先
声明顺序优先 同一路径下，先声明的优先
直接依赖覆盖传递依赖 直接声明的依赖版本优先

2. 依赖排除

场景

当传递依赖导致冲突时，需要排除特定依赖。

常用场景

场景 说明
排除冲突依赖 解决jar包版本冲突
排除不需要的依赖 减少打包体积
替换实现 如用Jetty替换Tomcat

3. 依赖范围

范围 编译 测试 运行 说明
compile ✅ ✅ ✅ 默认，所有阶段可用
provided ✅ ✅ ❌ 运行时由容器提供（如Servlet API）
runtime ❌ ✅ ✅ 运行时需要（如JDBC驱动）
test ❌ ✅ ❌ 仅测试阶段可用（如JUnit）
system ✅ ✅ ❌ 系统依赖，需指定路径

四、面试高频问题

1. Maven依赖冲突如何解决？

答案：

  1. 使用mvn dependency:tree查看依赖树
  2. 找到冲突的依赖版本
  3. 在pom.xml中使用<exclusions>排除不需要的版本
  4. 或者直接声明需要的版本

2. Git如何回退到之前的版本？

答案：

  • git reset --soft <commit-id>：保留修改，回退commit
  • git reset --hard <commit-id>：丢弃所有修改，完全回退
  • git revert <commit-id>：创建新的commit撤销指定commit

3. 如何查看Linux系统中某个端口被哪个进程占用？

答案：

  • netstat -tlnp | grep <port>
  • ss -tlnp | grep <port>
  • lsof -i :<port>
