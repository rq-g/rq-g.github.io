2. 聚簇索引 vs 非聚簇索引（面试必备）

特性 聚簇索引 非聚簇索引
数据存储 叶子节点存储完整数据 叶子节点存储主键
查询次数 1次（回表） 2次（先查主键，再查数据）
插入速度 插入到正确位置（可能页分裂） 较快（只需写入索引）
更新速度 更新主键代价高 较快
适用场景 主键查询、范围查询 辅助索引查询

二、MySQL锁机制（项目实战）

1. 锁类型（面试必备）

锁类型 说明 适用场景
行锁 锁定单行数据 精准控制并发
间隙锁 锁定行之间的间隙 防止幻读
记录锁 锁定单条记录 精准控制并发
表锁 锁定整张表 大表操作
意向锁 表级锁，表示行锁 多粒度锁

2. 事务隔离级别（面试必备）

隔离级别 脏读 不可重复读 幻读 实现方式
READ UNCOMMITTED 可能 可能 可能 无
READ COMMITTED 不可能 可能 可能 MVCC
REPEATABLE READ 不可能 不可能 可能 MVCC + 间隙锁
SERIALIZABLE 不可能 不可能 不可能 锁

3. 项目中的锁问题（智慧矿山项目实战）

项目背景：

  • 车辆调度需要原子性（扣减库存、创建订单）
  • 高并发下出现死锁问题
  • 等待锁超时导致系统卡顿

项目数据：

  • 死锁频率：每天10次
  • 等待锁超时：每天100次
  • 解决方案：优化事务粒度 + 调整锁顺序
