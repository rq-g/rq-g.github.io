三、遍历 ArrayList 时能否直接删除元素

1. 问题演示

2. 为什么会出问题？

for循环问题：删除元素后，后面的元素前移，索引变化，导致元素被跳过。

forEach问题：forEach内部使用迭代器，迭代器维护了modCount（修改次数），集合直接remove会导致modCount和expectedModCount不一致，抛出异常。

3. 正确删除方式

方式1：迭代器 remove（推荐）

方式2：倒序遍历

方式3：removeIf（Java8+，最优雅）

方式4：Stream 过滤（不修改原集合）

4. 总结对比

方式 正确性 推荐度 说明
正序for + remove ❌ 错误 ⭐ 会漏掉元素
forEach + remove ❌ 错误 ⭐ 抛ConcurrentModificationException
迭代器remove ✅ 正确 ⭐⭐⭐ 最通用
倒序for + remove ✅ 正确 ⭐⭐ 可行但不够优雅
removeIf ✅ 正确 ⭐⭐⭐⭐⭐ Java8+最优雅
Stream过滤 ✅ 正确 ⭐⭐⭐⭐ 不修改原集合

五、最佳实践总结

场景 最佳实践
循环字符串拼接 用StringBuilder，预分配容量
集合元素拼接 用StringJoiner或Collectors.joining()
接口返回集合 永远返回空集合，不返回null
遍历删除元素 用迭代器remove或removeIf
包装类使用 先判空再拆箱，或用Optional

总结

这三个场景题都是实际开发中高频踩坑的点，面试时不仅要说出”不能这么做”，还要说出”为什么不能”和”应该怎么做”。记住：踩过的坑越多，面试时越有话说。

2022年我踩过这些坑后，写代码时会刻意注意：

  • 循环拼接字符串一定用StringBuilder
  • 接口返回集合一定返回空集合
  • 遍历删除一定用迭代器或removeIf

这些习惯让我避免了无数次线上问题。
